# SenLink (センリンク)

## 概要

SenLinkは、専門学校における就職活動支援の課題（情報の断片化、教員の事務負担、学生のタスク管理不足）を解決するための、就職活動支援プラットフォームです。

## プロジェクトの目標

本プロジェクトは、過去の「動けば良いという品質管理を欠いた開発」への反省から、実務レベルの開発を追求することを目的としています。

- **設計**: モジュラーモノリスを採用し、将来の拡張性と現在の開発効率を両立。
- **品質の担保**: テスト駆動開発（TDD）を基本とし、アーキテクチャテストによる「依存関係の強制」を実施。
- **基盤の堅牢化**: 監査ログの自動記録や、グローバルな例外処理・バリデーションの統一。

---

## アーキテクチャ：

- **ドメイン駆動の徹底**: `api / worker → domain ← infra` の依存関係を厳守。技術基盤（DBやメッセージング）にビジネスロジックが汚染されないよう設計しています。
- **自動監査ログ (Auto-Audit)**: Entity Framework Core のインターセプターと RabbitMQ を連携。サービス層にログのコードを書くことなく、DBの全変更を非同期に記録する基盤を構築済みです。
- **ロールベースの認可 (RBAC)**: JWTクレームを用いた属性ベースのアクセス制御を実施。学生・教員・管理者の権限分離をコントローラーレベルで徹底しています。

### モジュール構成
- **Auth**: 多要素認証（OTP）、JWT発行、認証基盤　※実装済み
- **School**: 学科・クラス管理、学生・教員プロフィール管理（オンボーディング） ※実装済み
- **Audit**: システム操作の非同期監査ログ基盤（Infrastructure連動） ※実装済み
- **Job / Activity / Request / Notification / Maintenance**: 求人、活動履歴、申請、通知、メンテナンス（順次実装中）

## 開発手法

### テスト駆動開発 (TDD)
すべてのビジネスロジックは **Red-Green-Refactor** のサイクルを経て実装されています。

### ドキュメント・ルール駆動
`docs/` 配下にて要件定義、DB設計、API定義を詳細に管理しています。
- **実装規約の明文化**: DTOの分割、バリデーション配置、監査ログ重複防止などのルールを定め、複数人開発やAI駆動開発においても一貫したコード品質を維持しています。
