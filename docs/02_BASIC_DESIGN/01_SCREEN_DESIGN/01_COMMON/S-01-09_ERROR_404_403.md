# S-01-09 エラー（404／403）

## 1. 概要
- 対象ロール：ALL
- 目的：権限外／存在しないページへのアクセス時に、適切なエラー案内を行う。
- 遷移元：
  - 存在しないURLへのアクセス（404）
  - 権限外アクセス（403相当）※運用方針により404表示で統一する場合あり
- 遷移先：
  - ログインへ：S-01-02（未ログインの場合）
  - 各ロールのホームへ（ログイン済みの場合）
    - Student → S-02-01
    - Teacher → S-03-01
    - Admin → S-04-01

## 2. 画面構成
- ヘッダー：
  - ログイン済み：あり（ロール別ヘッダー、共通部品定義に従う）
  - 未ログイン：非表示（または、ロゴのみ）
- メイン領域：
  - エラー種別表示（404／アクセス不可）
  - 案内文（原因の可能性）
  - 戻り導線（ホーム／ログイン）
- フッター：あり

## 3. 表示項目
- タイトル（例）
  - 404：ページが見つかりません
  - 403相当：アクセスできません
- 本文（例）
  - URLが間違っている、またはページが削除された可能性があります。
  - 権限が不足している可能性があります。
- ボタン
  - ログインへ（未ログイン時）
  - ホームへ（ログイン済み時）
- エラーメッセージ（概要レベルで可）
  - 学外アクセス（学内IP制限）：S-01-01へ（本画面ではなくS-01-01優先）

## 4. 操作
- ホームへ戻る
  - ロールに応じたホームへ遷移（S-02-01／S-03-01／S-04-01）
- ログインへ
  - S-01-02へ遷移（未ログインの場合）
- 再試行（任意）
  - ページリロード（必要なら）

## 5. 関連API
- なし

## 6. 補足
- 権限外アクセス時の表示方針は以下のいずれかで運用する。
  - A）403相当でも404表示に統一（情報漏えい抑止）
  - B）403は403として表示（ユーザーに理由を伝える）
  ※ 本設計ではAを推奨しつつ、実装で選択可能とする。
- 学外アクセスはApacheで遮断される前提のため、本画面ではなくS-01-01を表示する。
