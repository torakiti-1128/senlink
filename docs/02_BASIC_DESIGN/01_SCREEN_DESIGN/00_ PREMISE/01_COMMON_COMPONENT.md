# 共通部品定義（画面設計の共通レイアウト）
> 目的：各画面設計での説明を省略するため、共通で使う「画面部品」をここで定義する。  
> 対象：S-02（学生）/ S-03（教員）/ S-04（管理者）画面を中心に適用。S-01（認証・エラー系）は例外あり。

---

## 1. 適用範囲（表示有無）

### 1-1. ヘッダー
- 原則表示：ログイン後画面（S-02 / S-03 / S-04）
- 原則非表示：ログイン前画面（S-01-02〜S-01-06）
- 例外：
  - S-01-01（アクセス不可）：ロゴのみ（リンク無し）を表示してよい
  - S-01-09（エラー404/403）：ログイン済みなら通常ヘッダー、未ログインならロゴのみ

### 1-2. フッター
- 原則表示：全画面
- 例外：画面の視認性を優先する場合（例：ログイン系で非表示にしたいなど）は画面設計側で明記

---

## 2. 共通部品一覧（名称と責務）

| 部品名 | 種別 | 対象 | 責務 |
|---|---|---|---|
| Header（ロール別） | 共通 | S-02/S-03/S-04 | ロゴ表示、ロール別ナビ、通知、ユーザーメニュー |
| Footer | 共通 | 原則全画面 | コピーライト |
| Role Navigation | ロール別 | S-02/S-03/S-04 | 学生/教員/管理者それぞれの主要画面への導線 |
| Notification Entry | 共通 | S-02/S-03/S-04 | 通知一覧への導線、未読の視認 |
| User Menu | 共通 | S-02/S-03/S-04 | プロフィール/ログアウト等の共通操作 |
| Responsive Menu | 共通 | S-02/S-03/S-04 | モバイル時のメニュー格納（ハンバーガー） |

---

## 3. ヘッダー（共通仕様）

### 3-1. 表示要素（共通）
- 左：SenLinkロゴ
  - クリック時の遷移先：各ロールのホームへ
    - Student → S-02-01
    - Teacher → S-03-01
    - Admin → S-04-01
- 中：ロール別ナビ（PC表示時）
- 右：通知アイコン、ユーザーメニュー（ログイン後のみ）

### 3-2. 右側要素（共通）
- 通知アイコン
  - 未読がある場合：バッジ表示（件数）
  - 押下：S-01-07（通知一覧）へ
- ユーザーメニュー（ドロップダウン）
  - プロフィール：ロールに応じたプロフィール画面へ
    - Student → S-02-13
    - Teacher → S-03-09
    - Admin → S-03-09（管理者は教員ロールも持つ前提）
  - ログアウト：ログアウト実行 → ログイン画面（S-01-02）へ

### 3-3. ロール別ナビ（PC表示時）
- Student ナビ
  - ホーム（S-02-01）
  - 求人（S-02-02）
  - 応募（S-02-05）
  - ToDo（S-02-07）
  - 申請（S-02-09）
- Teacher ナビ
  - ホーム（S-03-01）
  - 学生（S-03-02）
  - 申請（S-03-04）
  - 求人管理（S-03-06）
- Admin ナビ
  - ホーム（S-04-01）
  - 権限管理（S-04-02）
  - クラス管理（S-04-03）
  - マスタ管理（S-04-04）
  - 設定管理（S-04-05）
  - システム監視（S-04-06）
  - （任意）教員機能への導線
    - 方針：Adminは教員機能も利用可能とする場合、Teacherナビ相当を「切替（ドロップダウン/タブ）」で表示してよい

※ ナビ項目の表示可否は「ロール（学生/教員/管理者）」で制御する。

---

## 4. レスポンシブ（ヘッダーの振る舞い）

### 4-1. PC表示（横幅が十分な場合）
- ロゴ + ロール別ナビ（横並び） + 通知 + ユーザーメニュー

### 4-2. モバイル表示（横幅が狭い場合）
- ロゴ（左） + 通知（右） + ハンバーガー（右）
- ハンバーガー内に「ロール別ナビ」を格納
- ユーザーメニュー（プロフィール/ログアウト）もハンバーガー内に格納してよい

---

## 5. フッター（共通仕様）

### 5-1. 表示要素
- コピーライト（必須）：例）© SenLink

---

## 6. 共通の遷移ルール（画面設計の省略用）

### 6-1. 認証・権限
- 未ログインでログイン後画面にアクセス → S-01-02（ログイン）へ
- 権限外アクセス → S-01-09（404/403）へ（原則404扱い）
- 学外アクセス（学内IP制限） → S-01-01（アクセス不可）へ

### 6-2. 通知
- 通知アイコン → S-01-07（通知一覧）
- 通知一覧の行選択 → S-01-08（通知詳細）

---

## 7. 補足（運用メモ）
- Adminは「管理者ロール」＋「教員ロール」を持つ前提とする（プロフィール遷移は教員プロフィールへ統一）。
- Adminに教員機能の導線を出す場合は、ヘッダー内に「Teacher機能切替」を設ける（表示方式は実装側で決定）。
- 本ファイルで定義していないUI（色・余白・コンポーネント詳細）は、基本設計では扱わない。
