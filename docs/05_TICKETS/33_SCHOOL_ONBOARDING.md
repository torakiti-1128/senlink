# 【開発】School：初回プロフィール登録（オンボーディング）画面の実装

## 目的
新規登録直後のユーザーが、自身の属性（学生または教員）に応じた詳細情報を登録できるオンボーディング画面を実装し、APIと連携させる。これにより、システム利用に必要な基本プロフィールを完結させる。

## 作業内容
- [ ] **SF-01: 共通マスタデータ取得の実装**
    - 学科一覧 (`GET /api/v1/school/departments`) の取得と選択肢への反映
    - クラス一覧 (`GET /api/v1/school/classes`) の取得と選択肢への反映
- [ ] **SF-02: 学生向けオンボーディング画面の実装**
    - 氏名、カナ、学籍番号、学科、クラス、生年月日等の入力フォーム作成
    - `POST /api/v1/school/students/onboarding` との連携
- [ ] **SF-03: 教員向けオンボーディング画面の実装**
    - 氏名、カナ、役職、研究室等の入力フォーム作成
    - `POST /api/v1/school/teachers/onboarding` との連携
- [ ] **SF-04: ロールベースの遷移制御**
    - ログイン後のロール（Student/Teacher）に応じた画面表示の切り替え
    - 登録完了後のダッシュボードへのリダイレクト

## 完了条件
- [ ] 学生が自身の所属学科・クラスを選択してプロフィールを正常に登録できること
- [ ] 教員が自身の詳細情報を正常に登録できること
- [ ] バリデーションエラーが適切に表示されること
- [ ] 登録完了後、ステータスが更新されダッシュボードへ遷移すること
