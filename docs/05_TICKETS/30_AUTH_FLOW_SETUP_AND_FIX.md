# 【開発】Auth：認証フローの構築と不具合修正、およびフロントエンド基盤の実装

## 目的
認証フロー（ログイン、新規登録、OTP、パスワードリセット）を完結させ、モダンなフロントエンド（Next.js）との連携を確立する。また、システム全体の安定稼働のために起動不全の解消とリファクタリングを行う。

## 作業内容
- [x] **起動不全およびインフラ基盤の修正**
    - MassTransit を 8.3.0 にダウングレード（ライセンスエラーの解消）
    - Worker へのデータベース接続文字列の設定（user-secrets）
    - Supabase への `one_time_passwords` テーブルの追加（Migration）
- [x] **バックエンドのリファクタリングと機能強化**
    - `Program.cs` のサービス登録・パイプライン設定を拡張メソッドへ集約
    - `AuthController`, `SchoolController` の機能別分割
    - 全 API アクセスを記録する `AuditLogMiddleware` の実装
    - パスワードリセットフローを Guid から 6桁 OTP へ変更
    - エラーレスポンスの Operation 名を大文字（SCREAMING_SNAKE_CASE）に統一
- [x] **フロントエンド（SenLink.App）の構築**
    - Next.js 15 + Tailwind CSS 4 + shadcn/ui の導入
    - ロゴコンポーネントおよび認証共通 UI（ログイン、登録、OTP、パスワードリセット）の実装
    - Axios による API クライアントの実装と CORS 設定の追加
- [x] **認証フローの統合と修正**
    - ログイン時の OTP 不要化（パスワード認証のみでログイン完了）
    - パスワードリセット最終ステップでの Email 検証追加
    - CORS プリフライト（OPTIONS）および IP 制限の競合修正

## 完了条件
- [x] API および Worker が正常に起動し、RabbitMQ 経由でログが保存されること
- [x] ログイン成功時に JWT トークンが発行され、フロントエンドで保持されること
- [x] パスワードリセットおよび新規登録が OTP 経由で正常に完了すること
- [x] 全ての API レスポンスの Operation 名が一貫したフォーマットであること
- [x] バックエンドのコードがクリーンに整理され、拡張メソッドにより保守性が向上していること
