# プロジェクト憲章: SenLink (センリンク)

## 1. プロジェクト概要
専門学校における「学生と教員」「学生と企業」の繋がり（Link）を最適化し、就職活動における情報集約とプロセス管理を効率化するWebシステム。

## 2. 背景と目的
### 2.1 開発背景：学生時代の開発のリベンジ
本プロジェクトは、学生時代の卒業研究で作成したシステムの「再構築（Re-engineering）」である。過去のプロジェクトでは以下の4つの技術的・組織的課題により、継続的な運用・保守が困難な状態に陥った。
- **上流工程の欠如**: 要件定義・設計をスキップした「コードファースト」な開発による仕様の迷走。
- **設計の不在**: ビジネスロジックがViewに混在し、重複コードが散見される低い保守性。
- **品質管理の欠如**: テストコードが皆無であり、修正に伴う退行（デグレ）が頻発。
- **開発プロセスの欠如**: バージョン管理ルールやタスク管理がなく、変更履歴が追跡不能。

### 2.2 プロジェクトの目的
実務（運用保守・テスト工程）で得た「品質」と「プロセス」の重要性を証明し、メガベンチャー水準のエンジニアリング能力を示す。
- **技術的証明**: モジュラーモノリス構成、TDD（テスト駆動開発）、クリーンアーキテクチャの概念を取り入れた堅牢な設計。
- **遂行能力の証明**: チケット駆動開発、Gitフロー、CI/CD、ドキュメント管理の徹底。

## 3. 解決する課題
1. **情報の散在**: Teams、メール、紙媒体に分散している求人情報の集約。
2. **管理コストの高騰**: 教員による手動での進捗集計や、個別催促連絡の自動化。
3. **コミュニケーションの不透明化**: 添削依頼や面接調整の履歴を可視化し、言った・言わないの防止。
4. **学生の自走支援**: タスクの自動生成（ToDoリスト）による、活動漏れの防止。

## 4. システム範囲 (Scope)

### 4.1 学生向け機能
- **求人マネジメント**: 求人検索、応募、ブックマーク機能。
- **自動ToDo生成**: 応募企業ごとの選考ステップ（書類・面接等）を自動でタスク化。
- **選考プロセス管理**: 書類添削・面接予約・合否報告の一連のワークフロー。
- **通知システム**: 期限間近のタスクや新規求人のプッシュ通知（またはメール）。

### 4.2 教員・管理者向け機能
- **ダッシュボード**: 担当クラス全体の活動状況、内定率のリアルタイム可視化。
- **学生詳細管理**: 個別の選考履歴、添削依頼へのフィードバック。
- **マスタ管理**: 求人情報の登録、ユーザー権限（管理者/一般教員）の制御。

### 4.3 アウトオブスコープ (やらないこと)
- 就職後の給与振込や労務管理。
- 企業側が直接ログインして操作する機能（フェーズ1では教員による代行入力を想定）。

## 5. 成功指標 (Success Metrics)
- **Code Quality**: テストカバレッジ 80% 以上の維持。
- **Maintainability**: 静的解析ツールによる指摘ゼロ、およびドキュメントとの乖離ゼロ。
- **Reliability**: 全主要ユースケースに対するE2Eテストの自動化。

## 6. 技術スタック方針
- **Backend**: Go / Python (Django/FastAPI) 等、型安全または拡張性の高い言語。
- **Architecture**: モジュラーモノリス（将来的なマイクロサービス化を意識した境界定義）。
- **Infrastructure**: AWS または GCP を活用したコンテナベースの運用。
- **CI/CD**: GitHub Actions による自動テスト・自動デプロイ。
